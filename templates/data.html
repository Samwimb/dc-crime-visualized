Data: ETL

Since this project requires data from multiple sources, we used ETL to combine all the data. ETL refers to the process of extracting, transforming, and loading data into a new host source.

  1. Extract
    Data for this project came from 3 sources:
    • Crime: Open Data DC
    • Illumination: Percentage of Illumination
    • Weather: National Oceanic and Atmospheric Administration (NOAA)â€™s National Climatic Data Center (NCDC)


  2. Transform
    • Pandas and Jupyter Notebooks were used to transform the data
    • The crime file was filtered for incidents reported in 2018
    • For lunar data, we transposed the columns and built the dates based on the column names
    • Historical weather data is not available for free in APIs, so we downloaded a text file, saved it as csv and aggregated the data by day (originally by hour), obtaining the average, maximum and minimum for each day
    • Finally, all three files were combined in one dataset: added percent illumination, avg, max and min temperatures per day


  3. Load
    • Using sqlalchemy, one sqlite database was created
